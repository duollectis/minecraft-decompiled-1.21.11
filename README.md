# Minecraft 1.21.11 — Декомпилированный и рефакторированный исходный код

> ⚠️ **Важное юридическое уведомление:** Этот репозиторий содержит декомпилированный исходный код игры Minecraft версии 1.21.11, принадлежащей **Mojang Studios / Microsoft**. Код публикуется исключительно в **образовательных целях** для изучения архитектуры игры и разработки модификаций. Автор репозитория не претендует на авторство оригинального кода и не использует его в коммерческих целях.

## О проекте

Этот репозиторий представляет собой декомпилированный исходный код Minecraft 1.21.11, прошедший глубокий рефакторинг и документирование с помощью нейросетей (Claude Sonnet от Anthropic).

### Маппинги

Имена классов, методов и полей взяты из **[Yarn](https://github.com/FabricMC/yarn)** — открытого проекта маппингов сообщества Fabric. Yarn предоставляет человекочитаемые имена вместо обфусцированных имён Mojang.

Часть символов, которые не покрыты Yarn-маппингами, была переименована вручную с помощью нейросетей:
- Поля вида `field_XXXXX` переименованы в осмысленные имена на основе контекста использования
- Методы вида `method_XXXXX` переименованы по их поведению и роли в логике класса
- Магические числовые и строковые литералы частично вынесены в именованные константы (`private static final`) и enum-значения

### Что было сделано

- **Декомпиляция** — оригинальный байткод Minecraft был декомпилирован с использованием стандартных инструментов
- **Применение маппингов** — к декомпилированному коду применены Yarn-маппинги для восстановления читаемых имён
- **Ручное переименование** — оставшиеся `field_XXX` / `method_XXX` символы переименованы нейросетью по контексту
- **Проставление констант** — магические числа и строки частично заменены на именованные константы
- **Рефакторинг кода** — весь код (~6100 классов) приведён к современным стандартам Java 17+:
  - Применены принципы SOLID, KISS, DRY
  - Устранены "магические числа" — вынесены в константы и enum
  - Заменены вложенные `if` на Guard Clauses (ранний возврат)
  - Использованы современные конструкции: Stream API, Optional, switch-выражения, Records
  - Код приведён к единому стилю форматирования
  - Все классы покрыты Javadoc-комментариями на русском языке.

## ⚠️ Известные ограничения и возможные баги

Рефакторинг ~6100 классов выполнялся автоматически с помощью нейросети. Несмотря на то что игра **запускается и работает**, в коде могут присутствовать скрытые баги, внесённые в процессе рефакторинга.

### Исправленные критические баги

В ходе тестирования были обнаружены и исправлены следующие баги, возникшие после рефакторинга:

| # | Файл | Описание бага |
|---|------|---------------|
| 1 | `client/Mouse.java` | Кнопки мыши не работали в игре |
| 2 | `client/model/ModelPart.java` | Сломанный рендер entity и block entity (части тела разъезжались) |
| 3 | `resources/assets/minecraft/lang/en_us.json` | Отсутствующие переводы клавиш (показывался ключ вместо названия) |
| 4 | `util/dynamic/Codecs.java` — `createCodecForPairObject()` | Ошибка `Not a json array: 0.0` при создании мира |
| 5 | `util/dynamic/Codecs.java` — `nonEmptyEntryList()` | Краш `unbound tag` при загрузке рецептов |
| 6 | `network/codec/PacketCodecs.java` — `registryEntryList()` | Ошибка декодирования пакета `recipe_book_add` |
| 7 | `util/dynamic/Codecs.java` — `orElsePartial()` | `ClassCastException: Pair cannot be cast to DataResult` при загрузке чанков |

### Что может быть сломано

- **Редкие игровые механики** — рефакторинг мог изменить логику в редко используемых путях кода
- **Граничные случаи** — условия с инвертированной логикой или изменёнными операторами сравнения
- **Сетевой протокол** — тонкие баги в кодеках пакетов могут проявляться только при определённых условиях
- **Генерация мира** — сложные алгоритмы шума и биомов могут давать незначительно отличающиеся результаты

### Как сообщить о баге

Если вы обнаружили баг, предположительно внесённый рефакторингом, создайте Issue с:
1. Описанием проблемы и шагами воспроизведения
2. Стектрейсом из `run/debug/` (файлы `disconnect-*.txt`, `chunk-*.txt`)
3. Версией Java и ОС

## Использование

Репозиторий предназначен для:
- Изучения внутренней архитектуры Minecraft
- Разработки модификаций на базе Fabric API
- Исследования алгоритмов генерации мира, ИИ мобов и сетевого протокола

### ⚠️ Требования для запуска

> **Игра не будет корректно работать без папки `assets`.**

Папка `assets` содержит текстуры, звуки, модели, переводы и другие ресурсы Minecraft. Она **не включена в репозиторий** по юридическим причинам (авторские права Mojang).

Чтобы получить `assets`, необходимо:
1. Иметь лицензионную копию Minecraft Java Edition
2. Запустить официальный лаунчер Minecraft и дождаться загрузки версии 1.21.11
3. Указать в аргументах запуска путь к директории с ассетами.

Без `assets` игра запустится, но будет отображать розовые текстуры, отсутствующие звуки и пустые переводы.

### Сборка и запуск

```bash
# Сборка проекта
./gradlew build

```

Запуск напрямую через `java`, передав все необходимые аргументы:

```
java -jar <путь к jar файлу>
  --accessToken 0
  --version 1.21.11
  --username <игровой никнейм>
  --assetsDir ./assets
  --assetIndex 29
  --gameDir ./run
```

> Полный список всех аргументов запуска с описанием — в файле **[LAUNCH_ARGS.md](LAUNCH_ARGS.md)**.

| Аргумент | Описание |
|---|---|
| `--accessToken` | Лицензионный ключ. Можно вставить свой или оставить `0` |
| `--version` | Версия игры — `1.21.11` |
| `--username` | Игровой никнейм |
| `--assetsDir` | Путь к папке с ассетами (текстуры, звуки, переводы) |
| `--assetIndex` | Имя файла индекса ассетов из папки `assets/indexes/` (без расширения `.json`). Для Minecraft 1.21.11 это `29`. Несколько версий игры могут использовать один индекс — точное значение указано в поле `assetIndex.id` в JSON-манифесте версии |
| `--gameDir` | Рабочая директория игры (сохранения, настройки, логи) |
